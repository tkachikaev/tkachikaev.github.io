# timur.dev

Личный статический сайт разработчика игр. Сайт размещается на GitHub Pages и не требует сборки: достаточно HTML, CSS, JavaScript и папки с изображениями.

Главный проект сайта — **Bubi**, яркий 2D-платформер о зелёной птице.

## Что есть на сайте

- главная страница с проектами и описанием;
- русская и английская версии сайта;
- светлая и тёмная темы;
- адаптивное мобильное меню;
- отдельная страница проекта Bubi;
- галерея Bubi с полноэкранным просмотром изображений;
- плавные анимации и индикатор прокрутки.

## Структура

```text
.

├─ en.html                 # Главная страница на английском
├─ bubi.html               # Страница игры Bubi на русском
├─ bubi-en.html            # Страница игры Bubi на английском
├─ css/
│  ├─ style.css            # Базовая вёрстка, компоненты и адаптив
│  ├─ theme.css            # Светлая и тёмная темы
│  ├─ bubi.css             # Страница Bubi и галерея
│  └─ language.css         # Переключатель RU / EN
├─ js/
│  ├─ main.js              # Тема, навигация, меню, анимации
│  └─ bubi.js              # Полноэкранная галерея Bubi
└─ assets/
   ├─ avatar.png           # Фото автора
   ├─ favicon.svg          # Иконка сайта
   └─ gallery/             # Изображения проектов и Bubi
```

## Языки

Переключатель `RU / EN` находится в шапке всех страниц.

- Русская версия: `index.html` и `bubi.html`.
- Английская версия: `en.html` и `bubi-en.html`.

Это отдельные статические страницы, а не машинный перевод в браузере. При изменении текста на главной или на странице Bubi нужно обновлять соответствующую версию второго языка.

## Что менять

### Текст и ссылки

- Главная на русском — `index.html`.
- Главная на английском — `en.html`.
- Bubi на русском — `bubi.html`.
- Bubi на английском — `bubi-en.html`.
- Контакты, GitHub и ссылки в шапке/подвале находятся прямо в HTML-файлах.

### Цвета и темы

- Основные цвета тёмной темы — в начале `css/style.css` в блоке `:root`.
- Светлая тема — `css/theme.css`.
- Главный акцентный цвет: `--accent`.
- Цвет акцента для градиентов: `--accent2`.

### Изображения

Все изображения лежат в `assets/gallery/`.

- Главная картинка: `topimage.jpg`.
- Карточка и обложка Bubi: `bubi.jpg`.
- Нижний баннер: `campfire-forest-v1.jpg`.

Для замены изображения лучше использовать новое имя файла, например:

```text
campfire-forest-v2.jpg
```

После этого заменить путь к нему в нужных HTML-файлах. Так браузеры не будут показывать старую картинку из кэша.

## Галерея Bubi

Галерея находится в `bubi.html` и `bubi-en.html`, в блоке `.bubi-gallery`.

Чтобы добавить изображение:

1. Загрузить файл в `assets/gallery/`.
2. Скопировать один блок `<button class="bubi-gallery-item">`.
3. Поменять `data-full`, `data-caption`, `aria-label`, `src` и `alt`.
4. Добавить такой же элемент во вторую языковую версию с переводом подписи.

Пример:

```html
<button
  class="bubi-gallery-item"
  type="button"
  data-full="assets/gallery/new-location.jpg"
  data-caption="Новая локация Bubi"
  aria-label="Открыть: новая локация Bubi"
>
  <img
    src="assets/gallery/new-location.jpg"
    alt="Новая локация Bubi"
    loading="lazy"
    decoding="async"
  >
</button>
```

Галерея поддерживает:

- открытие изображения по клику;
- переход кнопками на экране;
- переход клавишами `←` и `→`;
- закрытие по `Esc`, кнопке `×` или клику за пределами изображения.

## Кэш CSS и JavaScript

В HTML-файлах стили и скрипты подключены с версией:

```html
<link rel="stylesheet" href="css/style.css?v=6">
<script src="js/main.js?v=6"></script>
```

После заметного изменения CSS или JavaScript увеличь номер версии, например `v=7`, во всех страницах, где подключён изменённый файл.

## Локальный просмотр

Сайт можно открыть напрямую через `index.html` в браузере.

Для более точной проверки лучше запустить простой локальный сервер из папки проекта:

```bash
python -m http.server 8000
```

После этого открыть:

```text
http://localhost:8000
```

## Публикация

Сайт публикуется через GitHub Pages.

Текущая схема:

1. Изменения попадают в ветку `main`.
2. GitHub Actions собирает статический артефакт сайта.
3. GitHub Pages публикует новую версию.

В `Settings → Pages` источником публикации должен быть выбран **GitHub Actions**.

Если сайт после деплоя выглядит по-старому:

- обновить страницу через `Ctrl + F5`;
- проверить, что увеличена версия `?v=` у изменённых CSS/JS-файлов;
- в GitHub открыть вкладку **Actions** и проверить статус последнего запуска.

## Резервные ветки

Перед крупными изменениями создаются снимки сайта в отдельных ветках. Сейчас доступны:

```text
backup-before-refactor-20260703
site-snapshot-before-bubi
```

Они позволяют быстро вернуться к рабочему состоянию, если какая-то крупная правка не подойдёт.
